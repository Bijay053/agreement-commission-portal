1. Enable CSRF protection

Add django.middleware.csrf.CsrfViewMiddleware back into MIDDLEWARE.

Keep it before middleware that assumes CSRF is already handled.

Make frontend send CSRF token on every POST, PUT, PATCH, and DELETE request.

Ensure authenticated browser requests fail without a valid CSRF token. Django recommends keeping CSRF middleware enabled and documents that it should be in MIDDLEWARE.

2. Add full security middleware and headers

Add django.middleware.clickjacking.XFrameOptionsMiddleware.

Configure:

SECURE_HSTS_SECONDS

SECURE_HSTS_INCLUDE_SUBDOMAINS = True

SECURE_HSTS_PRELOAD = True

SECURE_SSL_REDIRECT = True

SECURE_CONTENT_TYPE_NOSNIFF = True

X_FRAME_OPTIONS = "DENY"

SECURE_REFERRER_POLICY = "same-origin" or stricter if compatible

Add Content Security Policy middleware and configure CSP in settings. Django 6 documents built-in CSP middleware and SECURE_CSP / SECURE_CSP_REPORT_ONLY settings.
