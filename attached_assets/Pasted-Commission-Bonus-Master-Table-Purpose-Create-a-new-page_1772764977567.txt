Commission / Bonus Master Table
Purpose

Create a new page in the portal where users can view:

all provider commission records

all bonus records

agreement-linked commission details

bonus conditions and payout rules

This page should work like a master commission register across all agreements.

Navigation

Add new menu item under Navigation:

Commission Table
or

Commission & Bonus

Recommended placement:

Dashboard

Providers

Contacts

Agreements

Bonus fields
Bonus Type examples

Support cases like:

Case 1: Slab-based bonus

1–5 students = No bonus

5–10 students = AUD 100 per student

10–50 students = AUD 200 per student

51+ students = AUD 500 per student

Case 2: Country target bonus

10 students from 1 country = AUD 10,000 bonus

Case 3: Flat target bonus

5 students enrolled = AUD 5,000

20 students enrolled = AUD 20,000

Case 4: SALP-based / special target-based

Bonus based on SALP or agreed target structure

So bonus structure should not be a single plain text field only. It should support structured records.

Agreement linking

Every commission/bonus row must link back to its agreement.

So if a user clicks provider commission row, they can open the exact agreement where that rule was uploaded.

Status logic

Commission/bonus row status can be:

Active

Upcoming

Expired

Draft

Based on agreement status + effective dates.

Example:

effective_from in future → Upcoming

effective_to passed → Expired

agreement terminated → Inactive/Expired

Upload source logic

If commission/bonus is uploaded inside agreement, it should automatically appear in this global table.

That means:

Agreement page = source entry point

Commission Table page = master view page

No duplicate manual entry needed.

Important logic

Do not keep bonus only in free-text note format.

Instead, store:

bonus condition

target range

bonus amount

country condition

So later the system can filter and report properly.

Summary requirement for developer

Create a new Commission & Bonus master table page that:

automatically pulls all commission and bonus records from all agreements

supports filters, search, reset, export

links each row to the source agreement

separates commission and bonus clearly

supports slab, flat, target-based, country-based, and SALP-based bonuses

You can send this short version to developer:

Create a new “Commission Table” page in the portal. It should automatically show all commission and bonus data uploaded under every agreement in a master table view. Add filters for provider, country, territory, intake, agreement status, commission type, and bonus type. Add tabs for Commission and Bonus, a reset filter button, export option, and link each row back to the source agreement.
