1. Most Common Cause: X-Frame-Options Blocking the PDF

If the backend sends this header:

X-Frame-Options: DENY

or

X-Frame-Options: SAMEORIGIN

Chrome will block the PDF from loading inside an iframe viewer.

Fix

Allow the PDF to load in the viewer.

Backend should return:

X-Frame-Options: SAMEORIGIN

or remove the header for the PDF endpoint.

2. Content Security Policy (CSP) Blocking the Frame

If your server sends a strict CSP like:

Content-Security-Policy: frame-ancestors 'none'

Chrome blocks the document.

Fix

Allow the domain:

Content-Security-Policy: frame-ancestors 'self'
3. File Being Served as Attachment Instead of Inline

If the server sends:

Content-Disposition: attachment

Chrome will not display it in a viewer.

Fix

Use:

Content-Disposition: inline

Example response header:

Content-Type: application/pdf
Content-Disposition: inline
4. Secure Viewer Endpoint Not Returning the File Properly

Your viewer likely loads from something like:

/secure-viewer/{doc_id}

But if the API returns JSON or redirects, the viewer fails.

Developer should ensure the endpoint returns the binary PDF stream.

Example:

GET /api/secure-document/{id}

Content-Type: application/pdf
5. Chrome Blocking Because of Mixed Content

If your portal is:

https://replit.dev

but the PDF is loaded from:

http://

Chrome blocks it.

Fix

Serve PDF via HTTPS only.

6. Recommended Secure Viewer Method

Instead of embedding the raw PDF directly, use a viewer like:

PDF.js

Example:

/viewer.html?file=/api/secure-document/{id}

This prevents Chrome blocking and gives more control.

7. Developer Quick Fix Checklist

Ask developer to check:

PDF response headers:

Content-Type: application/pdf
Content-Disposition: inline

Remove blocking header:

X-Frame-Options: DENY

Allow iframe via CSP:

frame-ancestors 'self'

Ensure file loads via HTTPS

Ensure endpoint returns PDF binary, not JSON.

8. Temporary Quick Test

Open the PDF directly:

https://replit.dev/api/secure-document/{id}

If it opens normally → the problem is the iframe viewer.

If it does not open → the problem is the server headers.

9. Best Secure Implementation (Recommended)

For confidential agreements:

Use:

PDF.js secure viewer

watermark overlay

disable download

log document views

This avoids Chrome blocking and improves security.
