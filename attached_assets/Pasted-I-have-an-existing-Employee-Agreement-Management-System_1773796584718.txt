FEATURE 1 — E-SIGNATURE & SIGNING FLOW
═══════════════════════════════════════════

WHAT TO ADD IN BACKEND (Python FastAPI):

Add these new columns to the existing 
agreements table (use ALTER TABLE, do not 
recreate the table):
- signature_data (text)         ← base64 signature image
- signing_token (varchar, unique) ← secure UUID for signing link
- token_expires_at (timestamp)  ← 7 days from sent date
- signed_at (timestamp)         ← when employee signed
- signed_pdf_url (text)         ← path to signed PDF file

Add new API routes in a new file 
routers/signing.py:

POST /api/signing/send/{agreement_id}
- Generates a unique UUID signing token
- Saves token + expiry (7 days from now) to agreement row
- Updates agreement status to "sent"
- Triggers the email (see Feature 2)
- Returns success response

GET /api/signing/verify/{token}
- Checks if token exists in database
- Checks token has not expired
- Returns agreement details + employee name + 
  full agreement text for display
- If token invalid or expired return 400 error

POST /api/signing/submit/{token}
- Receives signature_data (base64 image string)
- Validates token is still valid
- Calls pdf_service to embed signature into 
  the existing agreement PDF
- Saves signed PDF to uploads/signed/ folder
- Updates agreement row:
  status = "signed"
  signed_at = now()
  signature_data = received base64
  signed_pdf_url = path to new signed PDF
- Triggers confirmation emails to both 
  employee and admin (see Feature 2)
- Returns success

ADD in services/pdf_service.py:

New function embed_signature_to_pdf():
- Takes existing agreement PDF file path
- Takes base64 signature image
- Uses reportlab or pypdf to open the PDF
- Goes to the last page (signature page)
- Embeds the signature image at the 
  signature line position
- Adds signed date text below signature
- Saves as a new file in uploads/signed/
- Returns the new file path

WHAT TO ADD IN FRONTEND (React):

Create a new file src/pages/SignAgreement.jsx
This is a PUBLIC page (no login required).
Route: /sign/:token

This page does the following:

On load:
- Calls GET /api/signing/verify/:token
- If invalid or expired shows an error message:
  "This signing link is invalid or has expired. 
  Please contact HR."
- If valid shows the signing page

Signing page layout:
- Company logo and name at top
- Heading: "Employment Agreement — Please Review and Sign"
- A scrollable box showing the full agreement text
- A checkbox: "I have read and understood this 
  agreement and agree to all terms and conditions"
- Below checkbox: A signature pad component 
  using react-signature-canvas library
  with label "Sign Here"
- Two buttons under the pad: 
  "Clear" to reset signature
  "Submit Signature" (disabled until checkbox 
  ticked AND signature drawn)
- On submit: POST to /api/signing/submit/:token
  with the signature canvas data as base64
- Show loading spinner while submitting
- On success show confirmation message:
  "Thank you [Name]. Your agreement has been 
  signed successfully. A copy has been sent 
  to your email."
- On error show: "Something went wrong. 
  Please try again or contact HR."

Install required package:
npm install react-signature-canvas
