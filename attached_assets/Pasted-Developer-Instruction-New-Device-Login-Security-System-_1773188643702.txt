Developer Instruction: New Device Login Security System
Objective

Implement a security feature that detects when a user logs in from a new device or browser and sends warning notifications to both the user and the admin.

System Logic
1. Generate Browser Fingerprint

When a user logs in, the frontend should generate a browser fingerprint and send it to the backend.

Fingerprint should include:

Browser name

Browser version

Operating system

Screen resolution

Timezone

Language

Device type

User agent

Unique fingerprint ID

Example:

fingerprint: 7f2b8c5e1a3d9c4f
2. Check Device in Database

Create a table:

user_devices

id	user_id	fingerprint	device_name	ip_address	last_login

System check:

IF fingerprint exists for that user
→ Allow login normally

IF fingerprint does not exist
→ Mark as new device

3. Actions for New Device Login

When a new device is detected:

Save the device in user_devices

Send security warning email to the user

Send alert email to admin

(Optional but recommended) Require OTP verification

4. Data to Record

For every login save:

User ID

Browser fingerprint

Device name

Browser

Operating system

IP address

Location (country/city)

Login time

Table example:

login_events

| id | user_id | ip | fingerprint | location | login_time |

5. Security Rules

Trigger security alerts when:

Login from new device

Login from new country

Login from suspicious IP

Multiple failed login attempts
