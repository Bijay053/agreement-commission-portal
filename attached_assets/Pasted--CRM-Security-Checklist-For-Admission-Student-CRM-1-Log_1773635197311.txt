🔐 CRM Security Checklist (For Admission / Student CRM)
1️⃣ Login & Authentication Security

Check the following:

☐ Two-Factor Authentication (2FA) enabled for admins and managers
☐ Password minimum 12 characters
☐ Password must include uppercase, lowercase, number, special character
☐ Account lock after 5 failed login attempts
☐ Login CAPTCHA enabled
☐ Session timeout after 30 minutes inactivity
☐ Password reset links expire within 24–48 hours
☐ Login allowed only via HTTPS

2️⃣ Role-Based Access Control (RBAC)

Check that users only see what they need.

☐ Admin has full system access
☐ Admission staff can only access student records
☐ Finance team only sees commission/payment modules
☐ Agents can only view their own students
☐ Users cannot access modules outside their role
☐ Sensitive actions require admin approval

3️⃣ Data Encryption

Sensitive data must be encrypted.

☐ CRM uses HTTPS SSL certificate
☐ Passport numbers encrypted in database
☐ Bank details encrypted
☐ Portal passwords encrypted
☐ Database backups encrypted
☐ Secrets stored in AWS Secrets Manager

4️⃣ File Upload Security

Students upload documents like passports and bank statements.

☐ Allow only PDF, JPG, PNG
☐ Block executable files (.exe, .js, .php)
☐ Scan uploaded files with antivirus
☐ Store files outside public server directory
☐ Generate unique file names
☐ Limit file size

5️⃣ Database Security

Check database protection.

☐ Database hosted in AWS RDS private network
☐ Database not publicly accessible
☐ Strong database password
☐ Automatic daily backup enabled
☐ Backup retention 7–30 days
☐ Database logs enabled

6️⃣ API Security

If your CRM uses APIs.

☐ API authentication required
☐ Use JWT or secure tokens
☐ API rate limiting enabled
☐ API input validation implemented
☐ Sensitive APIs protected by permission checks

7️⃣ Audit Logs

Every important action must be logged.

☐ Login attempts logged
☐ Student record access logged
☐ Portal password reveal logged
☐ File downloads logged
☐ Commission updates logged
☐ Agent account changes logged
☐ Logs include user, time, IP address

Example log:

User: Suman
Action: Viewed student passport
Student ID: STU8292
Date: 16 Mar 2026
IP: 103.xxx.xxx.xxx
8️⃣ Server Security (AWS)

Check infrastructure security.

☐ Server firewall enabled
☐ Only port 443 (HTTPS) open publicly
☐ SSH restricted by IP
☐ AWS Security Groups configured properly
☐ Server OS updated regularly
☐ Unused ports closed

9️⃣ Secrets Management

Sensitive credentials must not be stored in code.

☐ Database password in AWS Secrets Manager
☐ API keys stored securely
☐ Email SMTP credentials stored securely
☐ Portal login credentials stored securely
☐ No secrets inside GitHub repository

🔟 Backup & Disaster Recovery

Ensure data can be recovered.

☐ Daily database backup
☐ Weekly full system backup
☐ File storage backup (S3 versioning)
☐ Backup restore tested monthly
☐ Disaster recovery plan documented

1️⃣1️⃣ Browser Security

Add secure HTTP headers.

☐ Content-Security-Policy
☐ X-Frame-Options
☐ X-XSS-Protection
☐ Strict-Transport-Security
☐ Secure cookies enabled

1️⃣2️⃣ Vulnerability Testing

Run regular security scans.

☐ Test for SQL injection
☐ Test for XSS attacks
☐ Test for CSRF attacks
☐ Test API endpoints
☐ Check authentication bypass

Recommended tools:

OWASP ZAP

Burp Suite

1️⃣3️⃣ Monitoring & Alerts

Monitor suspicious activities.

☐ Login attempt alerts
☐ Server error alerts
☐ Database access alerts
☐ File upload anomaly alerts
☐ High traffic or DDoS alerts

Use:

Amazon CloudWatch

1️⃣4️⃣ Compliance & Data Protection

Because CRM handles passport and financial documents.

☐ Data access restricted
☐ Staff trained on data privacy
☐ Documents stored securely
☐ Data retention policy implemented
☐ Users can request data removal if required

1️⃣5️⃣ Internal Security Policies

Operational security.

☐ Staff cannot share passwords via WhatsApp
☐ Staff must use password manager
☐ Access removed immediately when staff leave
☐ Staff devices protected with antivirus
☐ Periodic security training conducted

⭐ Minimum Security Level You Should Achieve

For your CRM handling student documents, ensure at least:

✔ HTTPS encryption
✔ Role-based access control
✔ 2FA login
✔ Encrypted database fields
✔ Secure file uploads
✔ Audit logs
✔ AWS secret manager
✔ Daily backup
✔ Security testing
