Build a secure Portal Access Manager inside our CRM so the admission team can access multiple university/admission portals safely without sharing passwords in WhatsApp, email, or sheets.

System must be developed in Replit for testing and deployed in AWS for production.

1. Main Goal

We have many admission portals for different universities and partners. We need one central place inside CRM where authorized team members can:

view portal list

open portal URL

copy username

reveal password only if permission is granted

update credentials

track who viewed or changed access

manage access by role/team/country

This module must be secure and production-ready.

2. Tech Stack

Use our existing stack:

Frontend: Next.js

Backend: Django

Database: PostgreSQL

Development: Replit

Production Hosting: AWS

Secrets Storage: AWS Secrets Manager

Logs: CloudWatch or DB audit logs

3. Important Security Rules
Must follow

Do not store passwords in plain text in database

Do not return password in list API

Do not expose secrets in frontend code

Do not store portal passwords in browser local storage

Do not use Replit for permanent production secrets

All sensitive actions must create audit logs

Access must be controlled by role and permission

4. Module Name in CRM

Add a new menu/module:

Settings → Portal Access Manager

Inside this module create these sections:

Portal List

Add Portal

Access Groups / Permissions

Audit Logs

Password Rotation / Last Updated

6. Password Storage Logic
Very important

Do not save actual password directly in PostgreSQL.

Required approach

Store actual credentials in AWS Secrets Manager.

In PostgreSQL save only portal metadata such as:

id

portal_name

portal_url

username

aws_secret_name or aws_secret_arn

category

country

team

notes

status

created_by

updated_by

created_at

12. Replit Development Rules

Use Replit only for:

UI development

backend development

testing

staging/demo

Do not use Replit for:

final production database

permanent secret storage

Use dummy or test credentials only during development.

13. AWS Production Setup

Deploy live version in AWS.

Production recommended setup

Django backend in EC2 / ECS / Elastic Beanstalk

PostgreSQL in AWS RDS

Secrets in AWS Secrets Manager

Logs in CloudWatch

Optional static/media in S3

IAM Requirement

Application must have permission to:

read specific secrets from Secrets Manager

Do not allow unrestricted secret access.
